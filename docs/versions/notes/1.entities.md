# Translations
- So, what I need is to automate stuff with EntityExtensions bundle, in this case : Translations.

My options are to use (this is the modern approach) : 
- Interfaces
- Traits
- Attributes

What I would need (take an product entity example) :
- Make the EntityExtension translations reusable across all entities
- Make product translatable
- Add product translations
- Ship the ProductBundle with translations
- Make translations extendable by a plugin builder developer and by the final user of the plugin.

# Lets understand first what each option is creted for and what are its capabilities :

## Traits
- used for code reuse
- docs say it's for reuse of a method in multiple classes that each live in a different hierarchy.
## Interfaces
- specify what methods and properties a class must implement
- all methods must be public
## Attributes
- provide structured machine-readable metadata to classes/methods/functions/parameters/properties and constants.
