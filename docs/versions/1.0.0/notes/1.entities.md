# Lets understand first what each option is created for and what are its capabilities :

## Traits
- used for code reuse
- docs say it's for reuse of a method in multiple classes that each live in a different hierarchy.

## Interfaces
- specify what methods and properties a class must implement
- all methods must be public
- make different classes interchangable
- can't have private/protected
### Property hooks
```php
interface HasTitle
{
    public string $title { get; set; }  // must be publicly readable AND writable
}

class Article implements HasTitle
{
    public string $title = '';  // ✔ done — this satisfies the interface completely
}
```

## Attributes
- provide structured machine-readable metadata to classes/methods/functions/parameters/properties and constants.
- they don't do anything, they're just notes pinned to classes/properties/methods
- generally dataholder (declared readonly)
- we use the Reflection API to work with them (find them)

### Reflection API
- when PHP loads a class, it has a description in memory, the API is used to read that description
```php
$reflection = new ReflectionObject($actionHandler);

foreach ($reflection->getMethods() as $method) {
    $attributes = $method->getAttributes(SetUp::class);

    if (count($attributes) > 0) {
       ...
    }
}
```

## Abstraction
- an abstract class enforces all child classes to implement the abstract methods declared.
