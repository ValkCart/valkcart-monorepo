# Author vision
- this project started because complex needs for a store with products that have many sizes.
- I don't like the complexity that hits a beginner from the start when starting to develop in a Ecommerce CMS/Framework oriented to developers. That's why I want this project to be friendly to beginner developers and have complexity added when it's needed but not from the start.
- In a scale from 1 to 10, I don't want the project to be a strong 9 technical and 2 for documentation. I would rather go from 7 technicals / 7 documentation.

# Stack
- Symfony LAST stack is going to be used here : Live Components, Asset mapper, Stimulus, Turbo.
- I relate to some directions of what DHH (ruby on rails) founder is talking about.
